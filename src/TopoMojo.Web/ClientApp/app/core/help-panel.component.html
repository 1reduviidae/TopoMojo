Getting Started

1
  • Login to build simulation topologies.
  • An administrator can register you.
  • Initiate a password-reset to set your password.
2
  • Click 'Browse' to find your topology.
  • If making a new topology, click 'New' on the Browse page.
  • Click the 'Edit' icon on the topology.
  • If you do not see an 'Edit' button, ask the creator to add you to the topology.
3
  • In edit-mode, click on the Name or Description to change it.
  • In edit-mode, click on 'add-user-emails' to add your teammates to a new topology.
  • To link to a template, click 'Find Templates' and select the desired template(s).
4
  • Click on the template Name or Description to change it.
  • Click on the 'networks' value to add or change the network names for the template.
  • Templates are initially linked as 'external' templates.
  • You can deploy it with your lab, but cannot make changes to it.
  • To convert the 'external' template to a 'local' template, click the '?' next to 'external'.
  • (The template must have status 'initialize' to be able to clone it)
  • Click the red 'Trash' icon to delete the template. (It must not be 'deployed')
5
  • After cloning a template, you must 'initialize' the template before deploying it.
  • Click on 'Deploy' to create the virtual machine
  • Once deployed, use the Start, Stop, Revert, Save, Delete buttons to manage the vm.
  • (You cannot 'Save' an 'external' template.)
  • It is a good practice to shutdown the guest OS before Saving
  • Click the 'Console' button to connect to the virtual machine console.
6
  • Click the 'Settings' gear in the upper left corner of the Console to view console tools.
  • You can upload an iso file if necessary.
  • Ask an administrator to upload an iso that should be public or that is over 1GB.
  • You can change the network of the machine's first adapter to 'bridge-net' to reach the internet.
  • Do not leave the network set to 'bridge-net'. After downloading your resources, change it back to the desired network.
